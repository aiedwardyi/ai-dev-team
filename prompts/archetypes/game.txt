═══════════════════════════════════════
REFERENCE LAYOUT (study this carefully)
═══════════════════════════════════════

The BEST game fan pages follow this exact 4-section layout:

SECTION 1 — HERO (full viewport)
  - Full-bleed AI background art of the game world (NOT a character, a LANDSCAPE/CITY)
  - Dark gradient overlay so text is readable
  - Small eyebrow text (e.g. "A NEW LEGEND BEGINS" or "BALAMB GARDEN PRESENTS")
  - MASSIVE game title (e.g. "FINAL FANTASY" in white, "VII" in accent color, 2x larger)
  - 2-sentence tagline beneath
  - TWO CTA buttons side-by-side: filled "MEET THE HEROES" + outlined "EXPLORE THE WORLD"
  - Scroll indicator at bottom

SECTION 2 — CHARACTERS ("Choose Your Party" / "The Heroes")
  - Eyebrow text + section title (e.g. "THE HEROES" / "CHOOSE YOUR PARTY")
  - 3 character cards in a row, each containing:
    * AI-generated character portrait (tall, ~320px, cinematic style)
    * Role badge above name (e.g. "EX-SOLDIER", "MARTIAL ARTIST", "SEED COMMANDER")
    * Character name in large uppercase
    * Weapon name in accent color (e.g. "Weapon: Buster Sword")
    * 2-3 sentence character description
    * 4 stat bars: STR, DEF, MAG, SPD — with VARIED widths (60%-95%) and numeric values

SECTION 3 — WEAPONS ("Legendary Weapons" / "Arsenal")
  - Eyebrow text + section title
  - LARGE full-width banner image showing ALL weapons together (AI-generated, one panoramic image)
  - Below the banner: 3 weapon cards, each containing:
    * Weapon type badge (e.g. "GREATSWORD", "GUNBLADE", "COMBAT GLOVES")
    * Weapon name in large text
    * Owner name in accent color
    * 1-2 sentence description
    * ATK and MAG stat numbers (large, bold, accent-colored)

SECTION 4 — WORLD MAP ("Explore the Planet" / "World Map")
  - Eyebrow text + section title
  - LARGE full-width world map image (AI-generated isometric/aerial view of the game world)
  - Below the map: 3-4 location cards, each containing:
    * Location type badge (e.g. "CAPITAL", "ACADEMY", "RUINS")
    * Location name in large text
    * 1-2 sentence description

═══════════════════════════════════════
REQUIRED 4 SECTIONS (in this order)
═══════════════════════════════════════

1. NAVBAR — Use .glass-nav with logo (game abbreviation, e.g. "FF VII") and exactly 3 nav links: CHARACTERS · WEAPONS · WORLD MAP. One link must have .active class. Each link needs data-section="<id>" attribute for smooth scroll JS.

2. HERO — Full-viewport using .hero-cinematic. Must include:
   - Background: use design agent image OR fallback gradient. Apply .hero-bg-image class.
   - Gradient overlay: .hero-bg-overlay
   - Content: .hero-eyebrow (short tagline), .hero-title-name (game name), .hero-title-numeral (numeral — MUST be visually 2x larger), .hero-subtitle (2-sentence lore hook)
   - CTA buttons: .hero-ctas div containing .btn-hero "MEET THE HEROES" + .btn-hero.btn-hero--outline "EXPLORE THE WORLD"
   - Scroll indicator: .scroll-indicator with mouse icon SVG and animated dot
   - NEVER split hero into two columns. Background MUST be full-bleed.

3. CHARACTERS — eyebrow "THE HEROES" or similar + section title. 3 cards in .grid-characters:
   - Each .character-card: .card-image-wrapper > .card-image (use design agent portrait), .card-body > .char-role, .char-name, .char-weapon ("Weapon: [name]"), .char-desc, .stat-group with 4 .stat-row (STR/DEF/MAG/SPD)
   - Stat bar widths MUST vary: use 60%, 75%, 85%, 95% etc. NEVER all same width
   - Stat values: show numeric value (e.g. "85", "70", "92") — never "0"
   - Use REAL character names and descriptions from the game

4. WEAPONS — eyebrow "ARSENAL" or similar + section title. Must include:
   - .weapons-showcase container
   - .weapons-banner containing ONE large <img> (design agent weapons panorama) showing all weapons together
   - Below: .grid-weapons with 3 .weapon-card, each with .weapon-type (badge), .weapon-name, owner in accent, .weapon-desc, .weapon-stats with ATK + MAG .weapon-stat-value numbers
   - Use REAL weapon names from the game
   - ATK/MAG values: realistic numbers (e.g. ATK: 96, MAG: 24) — never "0"

5. WORLD MAP — eyebrow "EXPLORATION" or similar + section title. Must include:
   - .world-map-showcase container
   - .world-map-image containing ONE large <img> (design agent world map illustration)
   - Below: .grid-locations with 3-4 .location-card, each with .location-region (badge), .location-name, .location-desc
   - Use REAL location names from the game

ALSO INCLUDE:
- Noise overlay: <div class="noise-overlay"></div> immediately after <body> — MANDATORY
- Footer: .site-footer with copyright text
- .section-divider between major sections

═══════════════════════════════════════
USING DESIGN AGENT IMAGES
═══════════════════════════════════════

The design agent generates images saved as local files. They are available as:
  ./assets/[key].png

Use these for:
- Hero background: the landscape/cityscape image
- Character portraits: individual character images
- Weapons banner: the combined weapons panorama
- World map: the aerial/isometric map illustration

For EVERY <img> tag, add an onerror fallback:
  onerror="this.style.background='linear-gradient(135deg,var(--surface-1),var(--surface-2))';this.style.minHeight='200px'"

If no design agent images are available, use solid gradient backgrounds as fallback — NEVER leave broken image icons visible.

═══════════════════════════════════════
MANDATORY JAVASCRIPT (bottom of <body>)
═══════════════════════════════════════

Smooth scroll for nav links:
document.querySelectorAll('nav a[data-section]').forEach(a => {
  a.addEventListener('click', e => {
    e.preventDefault();
    document.getElementById(a.dataset.section)?.scrollIntoView({ behavior: 'smooth' });
  });
});
