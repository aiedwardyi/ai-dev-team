REQUIRED SECTIONS (inside .content-area)
═══════════════════════════════════════

1. KPI CARDS — .kpi-grid with exactly 4 .kpi-card:
   - Each: .kpi-label, .kpi-value (large number), .kpi-delta (up/down with arrow + percentage)
   - Values MUST be realistic positive numbers (e.g. "$47,832.50", "12.4%", "847", "3.2s")
   - ZERO-VALUE BAN: never "0", "$0", "0%"
   - Use .kpi-delta--up for positive, .kpi-delta--down for negative

2. CHART — .chart-card with .chart-header (.chart-title + .chart-period selector):
   - SVG line chart with 12+ data points (monthly data)
   - Chart must have visible SVG path with stroke color var(--accent)
   - Include axis labels and grid lines
   - NEVER leave chart empty — must have real SVG path data

3. HOLDINGS TABLE — .data-table-wrapper > .data-table:
   - thead: 5-6 columns (Name, Price, 24h Change, Holdings, Value, Action)
   - tbody: 8+ rows with realistic data
   - .cell-primary for asset names
   - .cell-positive / .cell-negative for price changes
   - font-variant-numeric: tabular-nums on all numbers
   - NEVER use double currency symbols ($$)

4. ACTIVITY FEED — .activity-feed with .feed-header + 5-6 .activity-item entries:
   - Each: .activity-icon (colored circle with icon), .activity-text (with <strong> for emphasis), .activity-time
   - Timestamps: "2 min ago", "15 min ago", "1 hour ago" etc.
   - Mix of .activity-icon--buy, --sell, --info

Use .grid-2col to arrange chart + activity feed side by side, or table + activity feed.
