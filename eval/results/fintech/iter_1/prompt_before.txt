# ARCHETYPE 5: FINTECH / TRADING

PATH: B (Raw CSS + animations)
Use for: stock trading platforms, banking apps, payment dashboards, investment portals, crypto exchanges.
Theme: Deep navy dark, professional, and data-focused. Base background: #0A0E1A. Card background: #111827. Surface background (e.g., modals, deeper sections): #1E2536. Border color: #2A3348. This is a NAVY-tinted dark theme, NOT a neutral gray. Primary accent: #10B981 (emerald-500) for positive values, primary actions. Danger: #EF4444 (red-500) for negative values, sell actions. Warning: #F59E0B (amber-500). Secondary accent: #81A6ED (a softer, desaturated blue) for non-critical highlights, subtle interactive elements, and focus states. Text primary: #F1F5F9. Text muted: #94A3B8. Text subtle: #6B7280.
Fonts: Headings (H1-H6) and prominent display text: 'Space Grotesk', sans-serif (weights 600, 700). Body text, labels, general UI: 'Inter', sans-serif (weights 400, 500, 600). ALL numerical values (prices, percentages, quantities, dates, chart labels, KPIs): 'JetBrains Mono', monospace (weights 400, 500). Ensure 'font-variant-numeric: tabular-nums;' is applied to ALL JetBrains Mono elements for perfect alignment and readability.

CRITICAL DIFFERENTIATION FROM DASHBOARD:
- **Top Navigation Bar:** A horizontal nav bar at the very top, spanning full width. Nav height: 64px. Logo (e.g., "FinTrack") on the far left, primary navigation links (e.g., Dashboard, Trade, Portfolio, Research) centered, and user profile/notifications/settings on the far right. This is the #1 structural difference from dashboard which uses a left sidebar. Background: #111827 with a subtle `backdrop-filter: blur(8px)` and `border-bottom: 1px solid #2A3348`. Nav links should have `color: #94A3B8;` and `hover: color: #F1F5F9; text-shadow: 0 0 5px rgba(241, 245, 249, 0.2);`.
- **Full-Width 3-Column Layout:** Below the top nav and ticker strip, the main content area is full-width with a primary 3-column grid.
    -   **Left Panel (280px wide):** Dedicated to Watchlist and potentially other quick summaries.
    -   **Center Panel (flex-grow: 2):** Contains the main trading chart and the order placement panel.
    -   **Right Panel (320px wide):** For Order Book, Trade History, or Key Metrics.
    -   Use `gap: 24px` between columns. The overall content area should have `padding: 24px`.
- **Advanced Candlestick Chart:** The primary chart MUST be a detailed candlestick chart. Show OHLC candles with distinct green (#10B981) and red (#EF4444) fills. Include volume bars below the main chart, also colored green/red. The chart MUST display at least 15-20 data points (candles). Axis labels should be clear, use 'JetBrains Mono', and represent realistic timeframes (e.g., specific dates or hours, not "Day 1"). Add interactive range selectors (1D, 1W, 1M, 1Y, ALL) above the chart with clear active states.
- **Integrated Order Panel:** Positioned directly below or to the right of the main chart. It MUST include:
    -   Buy/Sell toggle tabs with clear active states (green for Buy, red for Sell) and smooth `transition: background-color 0.2s ease-out;`.
    -   Order type selector (Market, Limit, Stop-Limit) with a dropdown.
    -   Input fields for Amount, Price (if Limit order), Shares. Input fields should have `background: #1E2536; border: 1px solid #2A3348; border-radius: 8px; color: #F1F5F9;` and a distinct `focus` state (`border-color: #81A6ED; box-shadow: 0 0 0 2px rgba(129, 166, 237, 0.2);`).
    -   A large, prominent action button (e.g., "Buy AAPL") using the primary accent color (#10B981) with a subtle `box-shadow: 0 4px 15px rgba(16, 185, 129, 0.3);` and `hover: transform: translateY(-1px); box-shadow: 0 6px 20px rgba(16, 185, 129, 0.4);`.
    -   Subtle key metrics related to the selected asset (P/E Ratio, Market Cap, Dividend Yield, 52-week High/Low) below the order form, using 'JetBrains Mono' for values.
- **Dynamic Watchlist with Sparklines:** The left panel contains a vertical list of ticker symbols. Each row MUST include:
    -   Ticker symbol (e.g., AAPL) in 'Space Grotesk' (600 weight) and company name in 'Inter' (400 weight, muted color).
    -   Current price and percentage change (24H) in 'JetBrains Mono', with green/red coloring for the change.
    -   A miniature sparkline SVG (approx. 80px wide, 24px high) showing the last 20-30 data points, rendered as a `<polyline>` with a subtle `linear-gradient` fill below the line (e.g., `linear-gradient(to top, rgba(16, 185, 129, 0.1), transparent)` for positive, `rgba(239, 68, 68, 0.1)` for negative). The sparkline line color should match the change (green for positive, red for negative).
    -   A subtle, icon-only "Add to Watchlist" or "Remove from Watchlist" button (e.g., a plus or star icon) using `color: #6B7280;` that becomes `color: #81A6ED;` on hover, visible only on hover of the row or as a small, muted icon.
- **Live Order Book Visualization:** The right panel MUST feature a two-column bid/ask order book.
    -   Bids (green, left-aligned) and Asks (red, right-aligned).
    -   Each row should show Price and Size, both using 'JetBrains Mono'.
    -   Include semi-transparent background bars (green for bids, red for asks) that visually represent depth, extending from the center outwards. The length of the bar should be proportional to the order size relative to the largest order. These bars should have `opacity: 0.2`.
    -   The order book should display at least 10-15 bid/ask levels.

Rules:
- **Typography Precision:** ALL numerical values (prices, percentages, quantities, dates, chart labels, KPIs) MUST use 'JetBrains Mono' with `font-variant-numeric: tabular-nums;`. No exceptions. Headings use 'Space Grotesk', body text uses 'Inter'. Ensure appropriate font sizes for hierarchy (e.g., large KPIs 36-48px, body text 14-16px, muted labels 12px).
- **Color Semantics:** Green (#10B981) for positive values/gains, buy actions. Red (#EF4444) for losses/negative values, sell actions. NEVER swap these. Use secondary accent (#81A6ED) for neutral interactive elements, subtle highlights, and focus states.
- **Sparkline Generation:** Every ticker in the watchlist MUST have a `<svg>` element containing a `<polyline>` with at least 20-30 data points. The SVG should be approximately `80px` wide and `24px` high. The line color should match the positive/negative change. Add a subtle `linear-gradient` fill below the polyline for visual appeal (e.g., `linear-gradient(to top, rgba(16, 185, 129, 0.1), transparent)` for positive, `linear-gradient(to top, rgba(239, 68, 68, 0.1), transparent)` for negative).
- **Card Styling & Depth:** Cards use `background: #111827; border: 1px solid #2A3348; border-radius: 12px; padding: 20px;`. Add a subtle inner shadow `box-shadow: inset 0 1px 0 rgba(255,255,255,0.05), 0 4px 15px rgba(0,0,0,0.2);` for depth. For interactive cards (e.g., portfolio summary items), include a subtle `box-shadow` on hover: `box-shadow: 0 0 0 1px #81A6ED, 0 8px 20px rgba(0,0,0,0.3);` and a slight `transform: translateY(-2px);`.
- **Interactive States:** All buttons, navigation links, input fields, and table rows MUST have distinct `hover` and `focus` states. Use `transition: all 0.2s ease-out;` for smooth visual feedback. Input fields should have a `border-color: #81A6ED; box-shadow: 0 0 0 2px rgba(129, 166, 237, 0.2);` on focus. Table rows should have a subtle `background-color` change on hover.
- **Micro-Animations:** Implement subtle micro-animations for data updates (e.g., price changes briefly flash a lighter green/red background or text color change), chart range selections, and button clicks. These should be quick and non-distracting.
- **Header Ticker Strip:** A slim, horizontal strip (height: 40px) immediately below the main navigation, showing 5-7 major symbols (e.g., AAPL, NVDA, TSLA, MSFT, GOOGL, AMZN) with their live price and percentage change. This strip MUST be visually distinct, with `background: #0A0E1A; border-bottom: 1px solid #2A3348;`. It should either smoothly scroll horizontally (if space is limited) or be static with clear, concise data. Each ticker item should have a small green/red arrow icon indicating direction.
- **Layout Geometry:** The main content area should have `padding: 24px` around the 3-column grid. Each card/section within the grid should have `padding: 20px`. The overall layout should feel spacious yet dense with information.
- **Anti-Pattern:** NEVER use generic, visually heavy "Add to Watchlist" buttons. Instead, use a small, muted icon (e.g., a plus sign or star) that becomes more prominent on hover, or a text link with an underline on hover.
- **Anti-Pattern:** Avoid empty charts or tables. Populate all data visualizations with realistic, varied data points (e.g., 15-20 candlestick data points, 10-15 order book entries, 5-7 watchlist items, 10-15 holdings). Ensure KPI values are large and impactful.

MUST include: Top navigation bar (64px height) → Header ticker strip (40px height) → Main 3-column layout (Left: Watchlist with sparklines; Center: Candlestick chart with 15-20 candles + Order panel; Right: Live Order Book with 10-15 bid/ask levels) → Portfolio summary row (using a bento-grid like layout for 3-4 key metrics with large 'JetBrains Mono' values) → Your Holdings table (at least 10 rows with asset, price, change, holdings, value) → Recent Trades table (at least 8-10 rows with time, asset, type, quantity, price, status) → Market News feed (at least 4-5 distinct news articles with source and timestamp).
