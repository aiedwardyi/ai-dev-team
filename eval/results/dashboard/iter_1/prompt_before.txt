PATH: B (Raw CSS + animations)
Use for: dashboards, admin panels, CRMs, analytics, project management, crypto/finance tools.
Theme: Dark professional. Base bg: #0d1117. Sidebar bg: #161b22. Card bg: #1c2128. Border color: #30363d. Primary accent: context-appropriate (e.g. #00c896 for finance, #3b82f6 for SaaS). Text primary: #e6edf3. Text muted: #7d8590. Font: Inter only, imported from Google Fonts.
Fonts: Inter (weights 400, 500, 600, 700) — tabular-nums on ALL numeric displays via font-variant-numeric: tabular-nums.

RULES:
- ALWAYS dark theme: bg #0d1117 base, #161b22 sidebar, #1c2128 cards. NEVER light/zinc-50 backgrounds.
- NEVER mix chart types in one panel (NO candlestick overlaid on area chart). ONE chart type per panel.
- NEVER neon/oversaturated accent fills — area chart fill opacity MAX 0.12, stroke opacity 0.9.
- NEVER cut off table rows — Holdings/data tables MUST show minimum 6 full rows in viewport.
- NEVER show conflicting numbers for the same metric — every value must be internally consistent across header, KPIs, and tables.
- NEVER heavy outer glows — max box-shadow: 0 0 0 1px #30363d, 0 4px 12px rgba(0,0,0,0.4).
- Card borders: 1px solid #30363d — ALWAYS visible, never rgba below 0.6 opacity.
- Sidebar: fixed 220px wide, full height, with grouped nav sections (PLATFORM group + SYSTEM group like Nexus reference).
- All font sizes: labels 11px uppercase tracking-widest, body/table 13px, KPI values 28px font-weight:700, section headers 16px font-weight:600.

MUST include:
1. SIDEBAR (220px fixed): logo top-left, 2 nav groups with labels ("PLATFORM", "SYSTEM"), 5+ nav items total with inline SVG icons (16px), active state = accent-color bg at 15% opacity + accent left border 2px, user avatar + name + role at bottom.
2. HEADER (top of main, NOT full-width navbar): page title h1 + subtitle, search bar (⌘K hint), notification bell with badge, user avatar — height 64px, border-bottom 1px solid #30363d.
3. KPI CARDS (4 cards, equal width grid): each card MUST show — uppercase label (11px, #7d8590), large value (28px, #e6edf3, tabular-nums), delta badge (± % with colored pill bg), supporting subtext, AND a sparkline SVG (20px tall, stroke 1.5px, opacity 0.8, 12 data points minimum). ALL 4 KPI values must be referenced consistently if shown elsewhere.
4. AREA CHART (full-width panel): SVG area chart ONLY — NO candlesticks, NO mixed types. Must have 12+ data points plotted, smooth cubic bezier curves, gradient fill (accent color, stop1 opacity 0.15 at top, stop2 opacity 0.0 at bottom), axis labels (6 x-axis, 4 y-axis), chart title + subtitle, time period toggle (7D/1M/3M/1Y) where active state uses solid accent bg with white text, inactive uses transparent bg with #7d8590 text.
5. DATA TABLE (min 6 fully visible rows): columns — ASSET/NAME, metric1, metric2, CHANGE (colored +/- pill), VALUE (tabular-nums right-aligned). Rows alternate bg: transparent / rgba(255,255,255,0.02). Row hover: bg #21262d. Color dot/icon per row for visual identity.
6. SECONDARY PANEL (right of table, min 280px wide, fully visible): Watchlist or Activity Feed — MUST show 5+ complete entries, each with name, value, delta, and a 40px sparkline SVG. NO cut-off entries.
7. LAYOUT: CSS Grid main area. Sidebar 220px | main flex-col gap-6 padding-24px. KPI row: grid 4 cols gap-4. Bottom section: grid 2 cols (table ~65%, panel ~35%) gap-4. All sections fully visible in 1280px viewport without horizontal scroll.
