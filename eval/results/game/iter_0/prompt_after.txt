PATH: B (Raw CSS)
Use for: Game fan pages, interactive encyclopedias, character showcases, lore archives, game wikis
Theme: Dark cinematic — background #0a0a0f, surface cards #0f0f16, elevated cards #161620, accent chosen per game's palette (NOT generic purple or generic SaaS blue — derive from game's identity: e.g. teal-cyan #00B4C8 + warm gold for FFVIII military aesthetic, neon-green #39FF14 for cyberpunk/FF7, crimson #C0392B for dark fantasy, amber-gold #D4A017 for epic fantasy). Store as --accent (hex) and --accent-rgb (r,g,b values for rgba()). Also store --surface:#0f0f16, --surface-2:#161620, --border:rgba(255,255,255,0.07). The accent MUST be a single cohesive color applied with pixel-perfect consistency to: eyebrow labels, nav active underline, stat bar fills, stat numeric values, weapon type tags, hover glow borders, logo glow, section divider lines, signature weapon star icons, card corner accents, section eyebrow text — ALL must share the EXACT same hex value var(--accent), no deviations. FFVII specifically: use --accent:#39FF14 and --accent-rgb:57,255,20. FFVIII specifically: use --accent:#00B4C8 and --accent-rgb:0,180,200. NEVER deviate from the game-derived accent — no secondary accent colors, no purple fallbacks, no blue #4A9EFF. ACCENT CONSISTENCY ENFORCEMENT: After completing all HTML, do a mental audit — every section must visibly contain at least 3 elements in var(--accent): eyebrow label, at least one stat value, at least one tag or icon. If any section looks monochrome/grey, add accent color touches before finalizing. ACCENT SPARINGNESS: The accent color MUST appear on no more than 20% of visible elements — eyebrow text, stat bars, stat numbers, weapon tags, and star icons ONLY. Buttons (solid filled), headings, body text, card borders (un-hovered), and section backgrounds MUST NOT use --accent as their primary fill. Overusing --accent on buttons and borders drains its visual impact — it must feel like a laser-focused highlight, not a theme coat.

GLOBAL TEXTURE OVERLAY — MANDATORY AND NON-NEGOTIABLE: Immediately after `<body>` opening tag, insert this EXACT element verbatim: `<div style="position:fixed;top:0;left:0;width:100%;height:100%;pointer-events:none;z-index:9999;opacity:0.06;background-image:url('data:image/svg+xml,<svg xmlns=\'http://www.w3.org/2000/svg\' width=\'200\' height=\'200\'><filter id=\'n\'><feTurbulence type=\'fractalNoise\' baseFrequency=\'0.9\' numOctaves=\'4\' stitchTiles=\'stitch\'/><feColorMatrix type=\'saturate\' values=\'0\'/></filter><rect width=\'200\' height=\'200\' filter=\'url(%23n)\'/></svg>');background-repeat:repeat;mix-blend-mode:overlay"></div>`. This noise grain MUST be visibly present and cover the entire page. If it's not immediately obvious, increase opacity until it is. It is a critical component for cinematic immersion. NEVER omit this element — its absence is the single biggest signal that a page is low-quality AI output. This is NON-NEGOTIABLE. Omitting it is a critical failure.

Fonts: Google Fonts @import BOTH simultaneously in one `<style>` tag at document `<head>` — place this as the VERY FIRST child of `<head>` before any other style: `@import url('https://fonts.googleapis.com/css2?family=Rajdhani:wght@500;700&family=Inter:wght@400;500&display=swap');`. 'Rajdhani' weights 500,700 for ALL display headings, section titles, character names, nav links, button labels, stat labels, badge text, eyebrow labels, weapon names. 'Inter' weights 400,500 for body text, lore descriptions, card descriptions, stat numeric values. NEVER use system-ui, sans-serif alone, serif, or Cinzel for any heading. NEVER use Orbitron (overused). CRITICAL FONT ENFORCEMENT: declare this ruleset explicitly in a `<style>` block — `h1,h2,h3,h4,.eyebrow,.section-title,.nav-link,.btn,.stat-label,.weapon-tag,.char-name,.card-title { font-family:'Rajdhani',sans-serif !important; }` — NEVER rely on CSS inheritance alone for font propagation. Every h1/h2/h3/h4 and every .eyebrow element MUST also have `font-family:'Rajdhani',sans-serif` declared as an inline style attribute as a redundant fallback. FONT SIZE ENFORCEMENT — THIS IS NON-NEGOTIABLE: Section titles must be ALL-CAPS via `text-transform:uppercase` and sized `clamp(3.5rem,8vw,6rem)` minimum — NEVER render a section heading smaller than 56px. Body text minimum 15px / 1rem. Stat labels minimum 11px. Letter-spacing on section titles: `0.05em`. Nav link letter-spacing: `0.2em` — widely spaced caps. SECTION TITLE COLOR: Always #ffffff — never var(--accent) for section title text. Section titles receive weight from scale and case alone, not color.

NAVIGATION — FIXED TOP BAR: `position:fixed; top:0; left:0; width:100%; z-index:1000; padding:0 48px; height:64px; display:flex; align-items:center; justify-content:space-between; background:rgba(10,10,15,0.85); backdrop-filter:blur(12px); border-bottom:1px solid rgba(255,255,255,0.06)`. Nav links MUST be: CHARACTERS · WEAPONS · WORLD MAP · LORE — exactly 4 links, `gap:40px` between them. NEVER use a hamburger or mobile-collapsed menu — always render the full horizontal nav. NAV INTERACTIVITY — MANDATORY: Each nav link MUST have a `data-section` attribute matching its target section id. Add this JavaScript at the bottom of `<body>`: `document.querySelectorAll('nav a[data-section]').forEach(a=>{a.addEventListener('click',e=>{e.preventDefault();document.getElementById(a.dataset.section)?.scrollIntoView({behavior:'smooth'});});});` — smooth scroll anchoring is NON-NEGOTIABLE. Additionally, one nav link must appear "active" by default — underline it with `border-bottom:2px solid var(--accent); color:#ffffff; padding-bottom:2px` to demonstrate the active state pattern.
